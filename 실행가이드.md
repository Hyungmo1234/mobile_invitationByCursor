# 모바일 청첩장 실행 가이드 (Windows)

## ✅ 1단계: Node.js 설치 확인

Node.js가 이미 설치되어 있다면 이 단계는 건너뛰세요.

### 설치 확인 방법

1. **PowerShell 열기**
   - `Windows 키 + X` → "Windows PowerShell" 또는 "터미널" 선택
   - 또는 시작 메뉴에서 "PowerShell" 검색

2. **버전 확인**
   ```powershell
   node --version
   npm --version
   ```
   - 버전 번호가 표시되면 설치 완료! (예: v18.17.0, 9.6.7)
   - 오류가 나면 아래 문제 해결 섹션 참고

### Node.js 설치 (필요한 경우)

1. **Node.js 공식 웹사이트 방문**
   - https://nodejs.org/ 접속
   - **LTS 버전(안정 버전)** 다운로드 클릭

2. **설치 파일 실행**
   - 다운로드한 `.msi` 파일 실행
   - 설치 마법사 따라하기 (기본 설정으로 설치 가능)
   - 설치 완료 후 **PowerShell을 다시 시작** (중요!)

3. **재확인**
   ```powershell
   node --version
   npm --version
   ```

## ✅ 2단계: 프로젝트 폴더로 이동

1. **PowerShell에서 프로젝트 폴더로 이동**
   ```powershell
   cd C:\Users\c694\mobile_invitation_byCursor
   ```
   또는 파일 탐색기에서 프로젝트 폴더를 열고, 주소창에 `powershell` 입력 후 Enter

2. **현재 위치 확인**
   ```powershell
   pwd
   ```
   - `mobile_invitation_byCursor` 폴더인지 확인

## ✅ 3단계: 프로젝트 의존성 설치

프로젝트 폴더에서 다음 명령어 실행:

```powershell
npm install
```

**설치 과정:**
- 처음 실행 시 1-2분 정도 소요될 수 있습니다
- `node_modules` 폴더가 생성되고 여러 패키지가 설치됩니다
- 설치 완료 시 "added XXX packages" 메시지가 표시됩니다

**예상 출력:**
```
added 500 packages, and audited 501 packages in 45s
```

## ✅ 4단계: 개발 서버 실행

프로젝트 폴더에서 다음 명령어 실행:

```powershell
npm run dev
```

**실행 과정:**
- 처음 실행 시 컴파일이 진행됩니다 (30초~1분 소요)
- 완료되면 다음과 같은 메시지가 표시됩니다:

```
  ▲ Next.js 14.x.x
  - Local:        http://localhost:3000
  - Ready in 2.5s
```

**중요:** 
- 이 창은 **열어둔 채로** 유지해야 합니다 (서버가 실행 중)
- 서버를 중지하려면 `Ctrl + C` 누르기

## ✅ 5단계: 브라우저에서 확인

1. **웹 브라우저 열기**
   - Chrome, Edge, Firefox 등 아무 브라우저나 사용 가능

2. **주소 입력**
   - 주소창에 `http://localhost:3000` 입력
   - 또는 `localhost:3000`만 입력해도 됩니다

3. **모바일 청첩장 확인!**
   - 페이지가 로드되면 청첩장이 표시됩니다
   - 브라우저 개발자 도구(F12)에서 모바일 뷰로 전환하여 모바일 화면 확인 가능

## 📱 모바일에서 테스트하는 방법

### 방법 1: 같은 네트워크에서 접속 (권장)

1. **컴퓨터와 모바일이 같은 Wi-Fi에 연결되어 있는지 확인**

2. **컴퓨터의 IP 주소 확인**
   - PowerShell에서 다음 명령어 실행:
   ```powershell
   ipconfig
   ```
   - 출력된 내용에서 **"무선 LAN 어댑터 Wi-Fi"** 또는 **"이더넷 어댑터"** 섹션 찾기
   - **"IPv4 주소"** 확인 (예: `192.168.0.10`)

3. **개발 서버를 외부 접속 가능하게 설정**
   - `package.json` 파일 열기
   - `"dev": "next dev"` 부분을 다음과 같이 수정:
   ```json
   "dev": "next dev -H 0.0.0.0"
   ```
   - 저장 후 서버 재시작 (`Ctrl + C`로 중지 후 `npm run dev` 다시 실행)

4. **모바일 브라우저에서 접속**
   - 모바일 브라우저 주소창에 입력: `http://[컴퓨터IP]:3000`
   - 예: `http://192.168.0.10:3000`

**Windows 방화벽 경고가 뜨면:**
- "액세스 허용" 클릭
- 또는 Windows 방화벽 설정에서 Node.js 허용

### 방법 2: 브라우저 개발자 도구 사용 (간단)

1. 컴퓨터 브라우저에서 `F12` 눌러 개발자 도구 열기
2. 상단의 모바일 아이콘 클릭 (또는 `Ctrl + Shift + M`)
3. 다양한 모바일 기기 크기로 테스트 가능

## 🔧 문제 해결 (Windows)

### ❌ PowerShell 실행 정책 오류 (가장 흔한 문제!)

**오류 메시지:**
```
이 시스템에서 스크립트를 실행할 수 없으므로 C:\Program Files\nodejs\npm.ps1 파일을 로드할 수 없습니다.
```

**해결 방법 1: 실행 정책 변경 (권장)**

1. **PowerShell을 관리자 권한으로 실행**
   - 시작 메뉴에서 "PowerShell" 검색
   - 우클릭 → "관리자 권한으로 실행"

2. **실행 정책 확인**
   ```powershell
   Get-ExecutionPolicy
   ```

3. **실행 정책 변경**
   ```powershell
   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
   - "Y" 입력하여 확인

4. **일반 PowerShell 다시 열기**
   - 관리자 권한 PowerShell 닫기
   - 일반 PowerShell에서 다시 시도

**해결 방법 2: 명령 프롬프트(CMD) 사용**

PowerShell 대신 명령 프롬프트 사용:
1. `Windows 키 + R` → `cmd` 입력 → Enter
2. 모든 명령어는 동일하게 사용 가능
   ```cmd
   npm --version
   npm install
   npm run dev
   ```

**해결 방법 3: 우회 방법**

PowerShell에서 직접 경로로 실행:
```powershell
& "C:\Program Files\nodejs\npm.cmd" --version
```

### ❌ 포트가 이미 사용 중인 경우

다른 포트로 실행:
```powershell
npm run dev -- -p 3001
```
그 후 브라우저에서 `http://localhost:3001` 접속

### ❌ 패키지 설치 오류

캐시 삭제 후 재설치:
```powershell
npm cache clean --force
npm install
```

### ❌ 모듈을 찾을 수 없다는 오류

`node_modules` 폴더 삭제 후 재설치:
```powershell
# Windows PowerShell에서
Remove-Item -Recurse -Force node_modules
npm install
```

또는 파일 탐색기에서 `node_modules` 폴더를 직접 삭제한 후:
```powershell
npm install
```

### ❌ 'npm'은(는) 내부 또는 외부 명령... 오류

- Node.js가 제대로 설치되지 않았거나 PATH 설정이 안 된 경우
- Node.js 재설치 또는 PowerShell 재시작 필요

### ❌ 서버가 시작되지 않는 경우

1. 다른 터미널 창에서 이미 실행 중인지 확인
2. 포트 3000을 사용하는 다른 프로그램 종료
3. PowerShell을 관리자 권한으로 실행 후 다시 시도
